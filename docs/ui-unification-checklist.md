# DataEditorY UI 统一清单

## 目标

- 把 DataEditorY 收口到同一套桌面编辑器风格。
- 让深色和浅色主题共享同一套视觉语义，而不是分别补丁。
- 把颜色、阴影、圆角、危险态、功能态从组件局部实现收回到全局 token。
- 在保留信息密度的前提下，减少“每个模块都像不同人写的”割裂感。

## 已完成的基础工作

- 已补齐卡片列表类型色的双主题变量。
- 已把后台任务状态条、脚本按钮、卡图按钮、危险按钮开始改为语义 token 驱动。
- 已把列表筛选浮层、输入框、分页按钮、顶栏菜单的部分圆角和阴影改为统一 token。
- 已把弹窗的遮罩、圆角、阴影、主次按钮、关闭按钮收口到统一 token。
- 已把标签栏和编辑器表单里的主要控件半径、focus ring、危险态、强调态改到共享变量。

## 第一阶段：先统一基础视觉语言

- [x] 统一全局主题变量入口。
- [x] 为深色 / 浅色主题建立同名语义变量。
- [x] 抽出 `shadow / danger / feature / control-radius / focus-ring` 这类基础 token。
- [x] 统一弹窗遮罩、弹窗圆角和弹窗阴影语义。
- [ ] 清理剩余组件里的硬编码十六进制颜色。
- [ ] 清理剩余组件里的硬编码阴影。
- [ ] 清理剩余组件里的 `4px / 8px / 12px / 999px` 这类重复圆角写法。

## 第二阶段：统一交互控件

- [ ] 所有主按钮统一一套高度、字重、圆角、hover、disabled 逻辑。
- [ ] 所有次按钮统一一套背景层级。
- [ ] 所有危险按钮统一使用 `state-danger-*` token。
- [ ] 所有输入框、下拉框、搜索框统一 focus ring 和边框强度。
- [ ] 所有 badge / pill / 状态条统一胶囊半径和字重。

## 第三阶段：统一模块级风格

- [ ] 顶栏、标签栏、菜单、弹出层统一为同一套“桌面工具栏”语言。
- [ ] 列表区的搜索、筛选、统计、分页统一为同一套数据工作台语言。
- [ ] 编辑器头部、表单、底部动作栏统一为同一套编辑器语言。
- [ ] 对话框统一标题层级、按钮排序、留白和阴影。
- [ ] Toast、悬浮状态、加载状态统一反馈样式。

## 第四阶段：统一语义配色

- [ ] 主视觉只保留一套产品主色，不让功能色抢主导。
- [ ] 脚本、卡图、AI 等功能色只作为辅助识别，不单独形成新视觉体系。
- [ ] 卡片类型色只用于辅助阅读，不参与主要结构分层。
- [ ] 深浅主题分别校正对比度，保证可读性一致。

## 第五阶段：代码层规则

- [ ] 新组件优先使用全局 token，不直接写局部硬编码颜色。
- [ ] 组件内优先引用语义 token，不直接引用业务色。
- [ ] 阴影优先使用 `--shadow-*`，危险态优先使用 `--state-danger-*`。
- [ ] 功能按钮优先使用 `--feature-*` 语义，而不是直接写十六进制。
- [ ] 同类控件样式优先抽到共享 token，再做局部微调。

## 建议优先处理的文件

- `src/lib/features/shell/components/AppTopBar.svelte`
- `src/lib/components/CardList.svelte`
- `src/lib/components/CardEditor.svelte`
- `src/lib/features/card-editor/components/CardEditorFooter.svelte`

## 执行顺序建议

1. 先收口 token。
2. 再统一顶栏 / 列表 / 编辑区三大主区域。
3. 然后处理所有弹窗和浮层。
4. 最后再做局部美化和颜色微调。

## 本次改造的设计原则

- 不追求“花”，优先追求稳定、清楚、像一个产品。
- 不让单个功能模块长出独立视觉宇宙。
- 不在组件里重复发明颜色系统。
- 可以保留功能差异，但必须共用同一套骨架。
