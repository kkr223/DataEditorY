# DataEditorY

DataEditorY 是一款跨平台桌面应用，用于创建和编辑 YGOPro 自定义卡片数据库（`.cdb` 文件）。支持中英文界面。

项目提供两个变体：

| 变体 | 功能 |
|------|------|
| `base` | CDB 编辑 + AI 助手 |
| `extra` | base + 制卡器与卡图编辑 |

## 界面概览

启动后，你会看到以下结构：

```
+-- 顶部栏 (全局命令：打开、保存、工具、切换主题等) ---+
|                                                        |
+-- CDB 标签页 (每个打开一个数据库) ---------------------+
|  卡片列表 (左侧)  |  当前卡片工作区 (右侧)             |
|  ┌──────────────┐  |  ┌─────┬──────┬──────┬────┐       |
|  │ 名称搜索框    │  |  │卡片  │脚本  │卡图* │AI* │       |
|  │ 高级筛选面板  │  |  ├─────┴──────┴──────┴────┤       |
|  │ 虚拟化卡片列表 │  |  │   当前卡片编辑区        │       |
|  │ (分页浏览)    │  |  │   (表单/Lua/卡图/对话)  │       |
|  └──────────────┘  |  └──────────────────────────┘       |
+----------------------------------------------------------+
```

> `*` 卡图为 extra 变体独有，AI 两个变体均包含。

- **顶部栏**：全局操作入口，切换主题
- **卡片列表（左侧）**：搜索、筛选、选择卡片
- **Surface 标签页（右侧）**：每个 CDB 标签页内部有四个子面板——卡片编辑、脚本、卡图（extra）、AI。当前卡片切换时，所有子面板同步更新

## 快速上手

### 打开数据库

1. 点击顶部 **打开** 按钮（或 `Ctrl + O`），选择 `.cdb` 文件
2. 也可以直接把 `.cdb` 文件拖入窗口
3. 首次打开后，该文件会出现在 **最近打开** 列表中

### 浏览和选择卡片

- 左侧列表会显示当前数据库的全部卡片
- 在名称搜索框中输入关键词，列表实时过滤
- 点击任意卡片，右侧切换到该卡的编辑界面

### 编辑卡片

- 修改任意字段（名称、效果文字、攻守等）
- 修改会暂存在草稿中，草稿状态会以视觉提示标明
- 点击 **保存**（`Ctrl + S`）将草稿写入数据库文件

### 新建卡片

按 `Ctrl + Shift + N` 创建一张空白卡片草稿，填写信息后保存即可。

### AI 助手

1. 切换到 AI Surface 标签页
2. 输入指令（如"为当前卡片生成 Lua 脚本"）
3. 点击生成，AI 会读取当前上下文并返回提案
4. 审阅提案内容，确认无误后应用

## 搜索与筛选

卡片列表左侧提供多层次的搜索能力。

### 快速搜索

| 搜索类型 | 用法 | 示例 |
|---------|------|------|
| 名称搜索 | 输入任意文本，模糊匹配卡片名称 | `青眼`、`Blue-Eyes` |
| 描述搜索 | 输入文本，模糊匹配效果文字 | `破坏对方场上` |
| ID/Alias 搜索 | 输入数字前缀，匹配密码或同名卡编号 | `473` 匹配所有以 473 开头的密码 |

### 高级筛选

点击筛选面板展开所有条件：

- 攻守/等级范围
- 属性、种族、类型（多选、组合）
- Setcode 筛选
- 规则表达式查询

### 规则表达式

一种简化的查询语言，适合做组合条件、数值比较和位掩码判断。

可用字段：`id`、`alias`、`atk`、`def`、`level`、`attribute`、`race`、`type`、`linkmarker`

运算：`and` `or` `not` `>` `<` `>=` `<=` `=` `!=` `contains`（可简写为 `&`）

```
示例：
  atk > def and level < 3
  id >= 473 and alias = 0
  attribute = dark and race = dragon
  type & effect and linkmarker & up
  type contains fusion or type contains synchro
```

### 图片文件夹筛选

1. 在高级筛选底部点击 **选择文件夹**，选择本地 `pics/` 目录
2. 数字文件名的图片（如 `89631139.jpg`）直接按密码匹配
3. 非数字文件名（如 `青眼白龙.jpg`）按卡片名称查找对应密码
4. 筛选结果只显示有图片的卡片

### 卡组文本筛选

1. 在高级筛选底部粘贴 YDK 格式文本，或点击 **导入 YDK** 选择 `.ydk`/`.txt` 文件
2. 筛选结果只显示卡组中包含的卡片

> 图片文件夹筛选和卡组文本筛选可以叠加使用，取两者交集。

### DEX 风格快捷搜索

在卡片编辑区底部有 **重置搜索 / 清空为新卡 / 搜索** 按钮。点击"清空为新卡"后，当前草稿中的名称、描述、攻守、类型、属性、种族、setcode 等基础信息会被带入搜索条件。

> 注意：数值字段遵循 YGOPro 约定，`-1` 用于匹配原始值为 `0` 的场景（攻击力、守备力、灵摆刻度等需要搜索 `0` 时应输入 `-1`）。

## 卡片编辑

### 编辑工作流

```
修改字段 → 草稿 (Draft) → 提交到工作副本 → 保存到 .cdb
```

1. 直接在表单中修改卡片字段
2. 修改会自动保存到前端草稿中
3. 切换卡片或保存数据库时，草稿自动提交到会话工作副本
4. 点击 **保存** 按钮，工作副本真正写回原始 `.cdb` 文件

如果草稿有未保存的修改，切换卡片时会弹出确认窗口。

### 批量字段编辑

1. 在顶部 **工具** 菜单中选择 **批量字段编辑**
2. 设置筛选条件（如所有魔法卡、所有攻击力 > 2000 的怪兽等）
3. 指定要修改的字段和新值
4. 预览影响范围，确认后执行

### 选中和管理

- 左侧列表支持多选（Ctrl/Shift 点击）
- 支持的批量操作：复制（`Ctrl + C`）、粘贴（`Ctrl + V`）、删除（`Ctrl + D`）
- 卡片分组功能用于标记和整理
- **从草稿搜索**和**筛选建库**按钮可从当前筛选结果创建新数据库

## Lua 脚本编辑

每个卡片可以关联一个 YGOPro Lua 脚本，位于 `<数据库目录>/script/c<卡片密码>.lua`。

### 打开与创建

- 切换到 **Script** 标签页
- 如果脚本已存在，直接打开
- 如果不存在，按设置中的模板创建
- 新建脚本的模板内容可在设置中自定义

### 编辑器功能

编辑器基于 Monaco（VS Code 同款引擎），提供以下功能：

| 功能 | 说明 |
|------|------|
| 语法高亮 | 完整的 Lua 语法着色 |
| 自动补全 | 函数名、常量、API 提示 |
| Hover 提示 | 悬停查看函数签名与文档 |
| 错误诊断 | 未定义变量、未使用变量等检查 |
| 内置手册 | `F9` 打开常量手册，`F10` 打开函数手册 |
| 自定义 Snippet | `snippets.json` 中可定义 `#` 前缀的快捷模板 |
| 行号显示 | 带行号的代码编辑区 |

### 代码图片导出

- 将当前脚本导出为带语法高亮的 PNG 图片
- 图片顶部附带卡片基础信息与效果文本
- 默认写入系统剪贴板
- 可在设置中开启"同时保存到本地"，额外保存到 `.cdb` 所在目录
- 存在选区时，导出按钮变为"导出选中代码"，只导出涉及的行并保留原始行号

### 辅助功能

- `Alt/Option` 键：按住时暂时隐藏函数说明浮窗，松开后恢复
- 脚本标签页与卡片绑定，切换卡片时自动切换对应的脚本
- 保存脚本（`Ctrl + S` 在脚本中操作时）保存的是 `.lua` 文件本身，不会触发数据库保存

### 字符串查看

支持 `aux.Stringid(id, n)` 对应文本的查看和编辑，方便处理多语言卡名和效果文本。

## 卡图编辑（extra 变体）

基于 `yugioh-card-ts` 库渲染卡图，所见即所得。

### 卡图预览

切换至 **Image** 标签页后，自动根据当前卡片的字段数据渲染卡图预览。

### 编辑卡图配置

| 配置项 | 说明 |
|--------|------|
| 卡框类型 | 普通、效果、融合、同调、超量、连接、灵摆等 |
| 卡图样式 | 边框、属性图标、星级等视觉元素的微调 |
| 属性/种族 | 与卡片数据联动，也可临时覆盖 |

### 导入与导出

- 导入卡图到 `<数据库目录>/pics/<卡片密码>.jpg`
- 导出当前卡图为 PNG 或 JPG

### 批量卡图导出

在顶部 **工具 → 批量卡图导出** 中，选择筛选条件和输出格式，批量导出整批卡片的卡图。

> 每张卡的卡图配置保存在 `.dey` 元数据中，不影响 `.cdb` 本身。

## AI 助手（extra 变体）

AI 工作台是内置的智能助手，可以读取当前环境（打开的 CDB、卡片数据、脚本、卡图配置），帮助你完成各种任务。

### 配置

在 **设置 → AI** 中配置：

- **API Key**：你的 API 密钥（加密存储）
- **Base URL**：兼容 OpenAI 的接口地址
- **模型**：如 `gpt-4o`、`deepseek-chat` 等
- **温度**：控制输出的随机性

### 内置技能

| 技能 | 说明 |
|------|------|
| **脚本生成** | 为卡片生成完整的官方 YGOPro Lua 脚本，参考已有相似脚本的结构和 API |
| **脚本测试** | 为生成或已有的 Lua 脚本编写 JSON 测试计划，由内置 runner 使用 `ygopro-jstest` 核心执行 |
| **稿件解析** | 输入自然语言描述的卡片设定，自动提取名称、效果、攻守、类型等字段 |
| **批量改卡** | 按条件批量修改卡片字段，或执行文本替换 |
| **卡图翻译** | 翻译卡片名称和效果文本到指定语言 |
| **审稿** | 审查卡片数据的完整性和数值合理性 |

### 提案机制

AI 的输出以 **提案（Proposal）** 形式呈现：

1. AI 生成修改方案
2. 你在界面上审阅提案的每一项变更
3. 逐项确认或一键全部应用
4. 确认后变更才真正写入

也可在设置中开启 **全自动模式**，AI 自动应用通过的提案，操作日志完整保留。

### @card 引用

在对话中使用 `@card` 提及当前已打开的卡片数据，AI 会读取该卡的全部字段信息。

### 对话持久化

AI 对话记录保存在 `.dey` 元数据中，关闭重开不会丢失。

## 打包与合并

### 打包（ZIP / YPK）

将当前数据库及同目录下的资源打包为可分发的格式：

- 自动收集 `pics/`、`script/` 等资源
- 支持 ZIP 和 YPK 两种格式
- 可在设置中自定义打包内容

### 合并 CDB

合并多个数据库来源为一个：

1. 添加多个 `.cdb` 文件，或从文件夹批量导入
2. 来源列表越靠后优先级越高，后面的覆盖前面的同密码卡片
3. 合并前分析：重复密码、最终卡片数、卡图覆盖情况、脚本覆盖计划
4. 可选择是否同时合并卡图与脚本文件
5. 合并完成后结果直接打开

## 资源文件与元数据

### 运行时资源

| 资源 | 说明 |
|------|------|
| `resources/strings/` | 字符串资源，支持目录聚合 |
| `resources/cover.jpg` | 封面图片 |
| `resources/lua-intel/` | Lua 编辑器智能提示数据（函数、常量、snippet） |
| `resources/yugioh-card/` | 卡图模板资产（仅 extra 变体） |

**字符串资源**支持目录聚合：程序会读取 `strings/` 目录下的所有有效文本文件，与内置资源合并使用。桌面版还会额外扫描程序目录及其 `resources/strings/`。文件按名称顺序合并，方便分模块维护。

**Lua 智能提示**的源文件位于 `static/resources/`，包括 `_functions.txt`、`constant.lua`、`snippets.json` 等。日常维护只需要修改这些源文件，构建时自动生成到 `src/lib/data/lua-intel/`。

### .dey 元数据

编辑器专用的运行状态保存在 `.dey/{cdb名称}.workspace.json` 中，放在 `.cdb` 同目录下，不影响数据库文件本身。

存储内容：

- UI 布局和筛选状态
- 卡片分组
- 卡图覆盖配置
- AI 对话、提案和操作日志
- 打开的脚本标签页
- 任务历史

## 安装与构建

### 下载预构建版本

从 [Releases](https://github.com/kkr223/DataEditorY/releases) 页面下载对应平台的安装包或 AppImage。

### 从源码构建

```bash
# 克隆仓库
git clone https://github.com/kkr223/DataEditorY.git
cd DataEditorY

# 安装依赖
bun install

# 前端开发模式
bun run dev              # 默认 extra 变体
bun run dev:base         # base 变体
bun run dev:extra        # extra 变体

# 桌面应用开发模式
bun run tauri:dev        # 默认 extra 变体
bun run tauri:dev:base   # base 变体
bun run tauri:dev:extra  # extra 变体

# 生产构建
bun run tauri:build:base     # 构建 base 变体
bun run tauri:build:extra    # 构建 extra 变体
bun run tauri:build:all      # 构建全部变体

# 类型检查
bun run check:base
bun run check:extra

# 运行测试
bun test
cargo test --manifest-path src-tauri/Cargo.toml
```

**环境要求**：

- Node.js 18+
- Bun（包管理器和运行环境）
- Rust 工具链
- Tauri 2 系统依赖（参考 [Tauri Prerequisites](https://v2.tauri.app/start/prerequisites/)）

桌面构建产物通常位于 `src-tauri/target/release/bundle/`。

### Linux / Arch Linux 注意事项

在较新的 Arch Linux 环境中，AppImage 可能出现启动白屏或中文显示异常。

白屏时尝试：
```bash
LD_PRELOAD=/usr/lib/libwayland-client.so ./DataEditorY*.AppImage
```

中文字体异常或缺字时：
```bash
sudo pacman -S noto-fonts noto-fonts-cjk
```
